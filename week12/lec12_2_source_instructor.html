Protocol v3 Lab — Designing Your Protocol v3
Duration: portion of 75-minute class session (the rest is lec12_2a) · Platform: Hokie AI (default thread for v3 design; lec12_2a uses custom agent thread for deployment) · Submission: Thread URL submitted in Canvas; agent segment lec12_2a is a separate Canvas submission · Lecture Mode: consolidation
Learning Objectives

By the end of this lecture, students will be able to:

  1. Take their Protocol v2 and produce a written Protocol v3 by working the three planning dimensions from lec12_1 (anticipation, decomposition, trade-off articulation) into specific protocol language.
  2. Locate the single clause in v2 where each of the three sophistication dimensions is weakest, and rewrite that clause to close the gap.
  3. Distinguish v3 sophistication from longer protocol text — v3's anticipation clause is shorter than v2's reactive equivalent when written precisely.
  4. Produce a Protocol v3 document that can be deployed to a custom agent in lec12_2a.
  5. Write a reflection that names a specific AI moment from the M block and reports what the design work surfaced about the v2-to-v3 upgrade.
📋 Note for Students

Today you write Protocol v3. This is not v2 with planning vocabulary sprinkled in. v3 is a working document that integrates the three planning dimensions you learned yesterday in lec12_1 — anticipation, decomposition, trade-off articulation — into specific clauses your custom agent will run. The test is whether a stranger reading v3 could predict the agent's behavior on a structured task. After the synthesis work in the M block, you transition to lec12_2a, deploy v3 to your custom agent, and run the lab. You return to this thread at the end of class to write your reflection.

◉ Instructor Note

This is the synthesis lab for Protocol v3 and the conceptual payoff for lec12_1. The most common failure mode at this stage is sprinkling: students who paste their v2 into the AI exchange and add a sentence about "anticipating contingencies" to the scope clause without actually rewriting the planning rule. The board work at [4:00] sets the discipline: v3 is rewriting specific clauses in operational language, not appending dimension labels to v2. The transition to lec12_2a is at [27:00] — students need v3 ready as deployable agent instructions before they go. R12.2.1 lives at the end, after they return from lec12_2a, so deployment exposure can inform how the reflection names what the upgrade actually did. The R block prompt is design-focused, not deployment-focused: it asks what the design exchange surfaced about the upgrade. The deployment-focused reflection is R12.2a.1 in the agent segment.

S12.2.2 Standard Prompt AI: respond
The synthesis scaffold for the M block. Use this to structure the exchange: 1. ANTICIPATION CLAUSE. Pull up the v2 clause closest to scope-or-task-framing. Ask the AI to probe it: which contingency in the kinds of tasks you give your agent is this clause written for, and which is it not written for? Draft a v3 anticipation clause that names a specific contingency and the signal that it is arriving. Read the v3 clause aloud — is it shorter and more specific than the v2 it replaces, or longer and vaguer? 2. DECOMPOSITION CLAUSES. Pull up the v2 planning rule. Identify a structured task your agent handles (a memo, a briefing, a comparison) and ask the AI: what are the sub-decisions inside that task? Draft v3 sub-decision rules — one per sub-decision. Read them aloud — do the rules have distinct content, or are they restatements of one global rule? 3. TRADE-OFF ARTICULATION CLAUSE. Identify two v2 guidelines that can pull in opposite directions for a realistic task. Ask the AI to name the conflict explicitly. Draft a v3 trade-off clause that names the competing guidelines and gives a resolution criterion (which takes priority and under what condition). By the end, you should have at least one rewritten v3 clause for each dimension. Together with the unchanged parts of v2, these become your Protocol v3 document.
