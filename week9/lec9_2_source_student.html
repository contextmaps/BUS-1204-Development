Peer Simulation — Think Like Another User
Duration: 75 minutes · Platform: Hokie AI (custom personal agent thread) · Submission: Thread URL submitted in Canvas before leaving class · Lecture Mode: consolidation
Learning Objectives

By the end of this lecture, students will be able to:

  1. Inhabit three distinct user perspectives and run scenarios from each perspective through a deployed custom agent.
  2. Observe how a deployed agent's behavior varies or remains consistent across user stances, even when the agent instructions do not change.
  3. Identify which protocol components (scope, planning rule, escalation) are tested by each user perspective.
  4. Articulate what "assuming the user thinks like the designer" costs in protocol design.
  5. Identify the single most consequential protocol weakness revealed by the three perspectives combined.
📋 Note for Students

Yesterday you watched your agent drift across repeated uses. Today you change the dimension: the agent stays the same, but the user changes. You will run your agent as three different users — a more careless user, a user with different ethical instincts, and an adversarial user — and observe what each perspective reveals about your protocol. The skill you are developing is not just about testing. It is about imagining users who are not you, which is the prerequisite for designing agents that work for people you have never met.

Yesterday you saw what happens when the same user runs the same agent four times. Today you see what happens when three different users run the same agent once each. The agent does not change. The user does. What each perspective reveals depends on what your protocol assumed about its user without saying so. Perspective 1 tests your defaults. Perspective 2 tests your values. Perspective 3 tests your boundaries. What you find is the raw material for Protocol v2.
