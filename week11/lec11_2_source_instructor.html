Week 11 · Lecture 2
Memory and Autonomy — How Much Does an Agent Decide?
Duration: 75 minutes · Platform: Hokie AI (default thread — reasoning partner) · Submission: Thread URL submitted in Canvas before leaving class · Lecture Mode: consolidation
Learning Objectives

By the end of this lecture, students will be able to:

  1. Distinguish between within-conversation memory and across-conversation memory and explain what each means for an agent's behavior.
  2. Explain how the autonomy spectrum applies to agents — from waiting for instructions to deciding and acting independently.
  3. Identify where their own custom agent sits on the autonomy spectrum based on its planning rule and escalation clause.
  4. Name at least one protocol-language implication of adding across-conversation memory to an agent.
  5. Connect memory and autonomy considerations to the upcoming team capstone agent design.
📋 Note for Students

Today you look at two dimensions that distinguish agents from each other: memory and autonomy. Memory is about what the agent carries from one conversation to the next. Autonomy is about how much the agent decides for itself versus waits for you to tell it what to do. Both dimensions are design choices — and both are expressed in protocol language. Today you think through what each one means for your own custom agent and for the team agent you will design starting next week.

◉ Instructor Note

This is the two-concept lecture. Memory and autonomy are not platform features students configure today — they are conceptual variables. The pedagogical risk is staying at abstraction: students who "get" both concepts without connecting them to specific protocol clauses have not arrived at the level of understanding this lecture targets. Circulate during M11.2.1 and push students who are describing the concepts correctly but generically. The key nudge: "What would your scope clause need to say if Study Helper remembered your assignment history?" The consolidation mode means students should leave with protocol-language insights, not just concept definitions. R11.2.1 is calibrated Satisfactory — this is intentional. The rubric distinguishes concept-level positioning (Satisfactory) from clause-grounded reasoning (Strong), which is the same family of distinction as synthesis vs. addition in Week 10.

Today we go one level deeper: we look at two dimensions that distinguish agents from each other. Same underlying LLM. Different protocol choices. Very different agent behavior."
  2. 4:00 Memory on the board. Write two columns: WITHIN-CONVERSATION MEMORY and ACROSS-CONVERSATION MEMORY. Within-conversation: the agent remembers everything said earlier in this thread. You have always had this. When you paste your protocol at the start of a session, the agent can refer to it later in the same conversation. Across-conversation: the agent remembers you when you open a new thread tomorrow. It might know you have been working on microeconomics all week. It might know you asked about the same paper three days ago. Not all platforms support this — and not all protocols are ready for it.
  3. 10:00 Autonomy on the board. Draw a horizontal spectrum: LOW AUTONOMY (asks before acting, surfaces options, defers decisions to the human) on the left. HIGH AUTONOMY (decides and acts, only escalates exceptions) on the right. Most agents land somewhere in the middle. What determines where? Two things: the planning rule (how does the agent decide what to do when a request is ambiguous?) and the escalation rule (when does it ask for help rather than just proceeding?). Those two components together set the autonomy level.
  4. 16:00 Paste R0. Students paste into default HokieAI thread.

  7. 55:00 Wrap M11.2.1. Brief in-class discussion: "Where does your custom agent sit on the autonomy spectrum? Show by hand — low autonomy on this side of the room, high autonomy on that side." Observe the distribution. Likely most students land toward low-medium. Note: "Most of you are in the low-autonomy range. That's a feature, not a limitation — low-autonomy agents ask before acting, which makes their errors easier to catch."
