Learning Objectives

By the end of this lecture, students will be able to:

  1. Integrate ethics additions from Week 7, operational tightening from Week 8, drift response from Week 9 lec9_1, user-variance awareness from Week 9 lec9_2, and failure prevention from lec10_1 into a single coherent Protocol v2.
  2. Distinguish between additive integration (v1 plus appended clauses) and genuine synthesis (restructured components with integrated language).
  3. Recognize where additions to Protocol v1 overlap, conflict, or reinforce each other, and consolidate them into fewer, more precise components.
  4. Produce a Protocol v2 that reads as a working document rather than a patchwork of accumulated notes.
  5. Articulate what kind of agentic system they want to build for the capstone, grounded in what v2 work revealed.
📋 Note for Students

Today you write Protocol v2. This is not Protocol v1 with additions bolted on. v2 is a complete rewrite: you take everything you have learned across Phase 2 and you produce a document that anticipates what v1 did not. The test is whether a stranger reading v2 would understand what the agent is, what it does, how it decides, and when it stops — without needing to read v1 first.

This is the synthesis lecture. Everything you have done in Phase 2 — writing Protocol v1, finding its gaps, adding ethics language, testing it in deployment, watching it drift, running it as three different users, diagnosing failure cases — all of it was preparation for this moment. Protocol v2 is not a collection of that work. It is what that work taught you, expressed as a working document.
