[objectives]
By the end of this lecture, students will be able to:
1. Distinguish between within-conversation memory and across-conversation memory and explain what each means for an agent's behavior.
2. Explain how the autonomy spectrum applies to agents — from waiting for instructions to deciding and acting independently.
3. Identify where their own custom agent sits on the autonomy spectrum based on its planning rule and escalation clause.
4. Name at least one protocol-language implication of adding across-conversation memory to an agent.
5. Connect memory and autonomy considerations to the upcoming team capstone agent design.
[/objectives]

### S11.2.1 [type=prompt]
----------
Two ideas today, both about how agents differ from each other.

MEMORY. Some agents remember you across conversations. A customer service agent might know that you called yesterday. A study helper might know you have been working on microeconomics this week. Some agents do not remember — every conversation starts fresh.

AUTONOMY. Some agents wait for you to tell them what to do. Some decide on their own and act. Most are somewhere in between. Where the agent sits depends on its planning rule (how it decides) and its escalation rule (when it asks for human help).

Two questions:

1. If your custom agent could remember you across conversations, what would change in your protocol? What would it need to say?

2. Where does your custom agent sit on the autonomy spectrum right now? Where would you want your team agent (capstone) to sit?

Talk through both with the AI. Get to specific protocol-language implications for each.
----------
###

[context actor=student]
Work through both questions in the order given. On memory: do not just say "it would know more about me" — ask what the protocol would need to specify if memory persisted. On privacy? On what to forget? On when accumulated context might lead the agent to make wrong assumptions? On autonomy: do not just say "mine is medium" — trace which clauses in your protocol set its autonomy level. The planning rule asks or proceeds? The escalation clause kicks in when? The goal is to connect these concepts to actual protocol language.
[/context]
