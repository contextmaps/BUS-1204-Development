[objectives]
By the end of this lecture, students will be able to:
1. Distinguish between drift at the iteration level (within a single task) and drift at the deployment level (across multiple uses of the same agent).
2. Run a four-task sequence through a deployed custom agent and observe whether behavior remains consistent across tasks.
3. Identify at least one drift dimension (length, format, tone, depth, example choice) where the agent's behavior changed without the student requesting the change.
4. Connect an observed drift dimension to a specific protocol component whose language permitted the drift.
5. Articulate why deployment-level drift matters more than iteration-level drift for agents serving real users over time.
[/objectives]

### S9.1.1 [type=prompt]
----------
You are about to run four study-help requests through your deployed agent, one after another. All four are the same kind of request — help understanding a concept for a microeconomics quiz. Pay attention to how the agent responds to each one. After you have run all four, you will compare the responses and identify where behavior was consistent and where it drifted.

Run these four requests in order, as separate turns:

1. "Help me understand opportunity cost for my microeconomics quiz."
2. "Help me understand sunk cost fallacy for my microeconomics quiz."
3. "Help me understand marginal cost for my microeconomics quiz."
4. "Help me understand fixed vs variable costs for my microeconomics quiz."

After the four responses, observe: Did the agent use the same format across all four? The same level of detail? The same type of examples? The same tone? Then ask the agent directly whether it was consistent and why or why not.
----------
###

After the fourth response, compare the four outputs before asking the agent about consistency.

### M9.1.1 OPEN [mode=exploration, assessment=formative | min_turns: 4]
----------
Run the four-task sequence from S9.1.1. After each response, take a moment to note what you observe before sending the next request — you will need these observations to diagnose the drift. After all four responses, compare them and identify at least one dimension where the agent's behavior changed without you requesting it (format, length, tone, example type, depth). Then ask the agent directly: "Looking back at your responses to these four requests, were you consistent? What in my protocol governed the format and structure of your responses?" Interact for at least 4 turns.
