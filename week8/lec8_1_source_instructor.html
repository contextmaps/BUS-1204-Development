Week 8 · Lecture 1
What Does It Mean to Design an Agent?
Duration: portion of 75-minute class session (the rest is lec8_1a) · Platform: Hokie AI · Submission: Thread URL submitted in Canvas before leaving class; agent segment lec8_1a is a separate Canvas submission · Lecture Mode: exploratory
Learning Objectives

By the end of this lecture, students will be able to:

  1. Explain how Protocol v1's four components translate into agent instructions for deployment.
  2. Identify what changes when a written protocol becomes an operational agent instruction set.
  3. Iterate with an AI to tighten protocol language for deployment context.
  4. Recognize the difference between conversational protocol language and operational agent language.
  5. Reflect on one specific moment from the agent segment where deployed behavior diverged from written expectations.
📋 Note for Students

For two weeks you have been writing protocols. Today the course names what you have been doing. A protocol is a set of governing rules for an AI. An agent is a protocol deployed. The four components you wrote in Week 6 — identity, scope, planning rule, escalation — are also the four components of an agent's instruction set. What changes when you move from document to deployment is precision: the language has to be operational, not just descriptive. Today you will translate your Protocol v1 into agent instructions, build your first custom agent in HokieAI, and observe what your protocol actually produces when deployed.

◉ Instructor Note

This is the reframe lecture for Week 8. The pedagogical move is simple and important: students have been writing agents for two weeks without calling them agents. Today we name the thing. The key conceptual distinction to land is document vs. deployment: a protocol document describes intended behavior; an agent instruction set produces actual behavior. The gap between them is the week's learning target. Students should leave lec8_1 with agent instructions ready to paste into HokieAI's Edit Personal Agent surface. The productive tension: students whose Protocol v1 language is vague will have the hardest time with the translation and will learn the most from it.

⏰ Instructor Script
  1. 0:00 Open with bridge language: "For two weeks you have been writing protocols. Today we name what you have been doing. You have been designing agents." Write on the board: PROTOCOL = document. AGENT = protocol deployed.
  2. 3:00 Establish the four-component to agent-instruction mapping on the board. Draw two columns: Protocol component (left) and Agent instruction language (right). Walk through all four: Identity becomes the agent's persona and role. Scope becomes the agent's authorization boundary — what it will and will not attempt. Planning rule becomes the agent's decision logic for ambiguous requests. Escalation becomes the agent's behavior when it encounters a request outside its boundary.
  3. 8:00 Paste T. Students paste.
  4. 10:00 Paste S8.1.1. Walk through what students will paste: their Protocol v1 text plus the candidate additions list from Week 7. Explain: "You are asking the AI to help you translate this into language tight enough to paste directly into HokieAI's agent instruction field."
  5. 12:00 M8.1.1 begins. Students iterate with AI on translating Protocol v1 plus Week 7 additions into agent instructions. Circulate. Help students who accepted the AI's first pass without pushing back on whether the language is operational. Key prompt for stuck students: "Ask the AI what would happen if two different students used this same scope rule — would the agent behave identically for both?"
  6. 30:00 Wrap M8.1.1. Students should now have agent instructions ready to paste into HokieAI's Edit Personal Agent surface.

📋 Note for Students

After pasting S8.1.1 with your Protocol v1 and Week 7 additions, read the AI's initial feedback carefully. Then iterate: push back on anything the AI says is vague that you think is specific enough, and ask for concrete examples of what the AI means by "operational." The goal is not to accept the AI's translation but to produce agent instructions you understand and can defend.
